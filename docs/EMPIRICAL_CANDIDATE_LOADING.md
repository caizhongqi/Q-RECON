# Explicit Empirical Candidate Loading Is Not a Free Oracle

## 1. Motivation

A real-data reconstruction experiment often starts from an explicit candidate
list

\[
T=(t_0,\ldots,t_{N-1}),\qquad t_i\in\{0,1\}^w,
\]

and then writes a Grover search over candidate indices. That search requires a
coherent lookup operation

\[
U_T|i\rangle|y\rangle
=|i\rangle|y\oplus t_i\rangle.
\]

A classical array or dataset file does not supply this unitary automatically.
The data must be compiled into a reversible circuit, stored in a declared QRAM
architecture, or generated by a succinct reversible procedure. Q-RECON therefore
separates the query complexity of the downstream verifier from the setup and
per-call cost of `U_T`.

This document formalizes two boundaries:

1. every exact compiler for an explicitly supplied arbitrary table has a linear
   worst-case input-reading cost;
2. under a finite bounded-arity gate library, almost all tables require circuits
   whose size is nearly linear in the table description length, up to the
   logarithm of the available gate locations.

Neither statement applies to a highly structured table with a short generator.
That distinction is precisely why Q-RECON prefers structure-preserving candidate
oracles over global truth-table enumeration.

## 2. Explicit-table model

Let `N` be the number of entries and `w` the output word width. The classical
description contains

\[
B=Nw
\]

bits. The address register has

\[
n_I=\lceil\log_2 N\rceil
\]

qubits. Unused binary addresses, when `N` is not a power of two, are defined to
return the all-zero word.

The clean lookup contract is

\[
U_T|i\rangle|y\rangle|0^a\rangle
=|i\rangle|y\oplus t_i\rangle|0^a\rangle.
\]

All work qubits must return to zero. Any candidate-state preparation and
reflection about that state are separate resources.

## 3. Compiler input-reading lower bound

### Theorem 1 — explicit-description probe lower bound

Any exact compiler whose input is the literal `B=Nw`-bit table description and
whose output implements `U_T` must inspect every description bit in the worst
case.

### Proof

Assume the compiler does not inspect one table bit corresponding to address `i`
and output position `j`. Consider two table descriptions that are equal on every
other bit and differ only at that position. The compiler follows the same
execution and emits the same artifact on both inputs. However, the required
lookup unitaries differ on the basis state with address `i` and target bit `j`.
One artifact cannot be exact for both tables, a contradiction. Therefore every
bit may be decisive and the worst-case compiler input cost is at least `B` bit
probes. \(\square\)

This lower bound is about explicit input processing. It applies even if the final
circuit happens to be compact. A claim that excludes this setup cost must state
that the table is preloaded, supplied by a separate oracle, or reused across a
declared number of reconstruction instances.

## 4. Circuit-counting lower bound

Fix a finite gate alphabet of size `g`, gates acting on at most `r` ordered wires,
and a total of `n` circuit qubits including ancillas. At one circuit position,
the number of concrete gate instances is at most

\[
A=g r n^r.
\]

There are `2^B` possible lookup tables. The number of circuits of length strictly
less than `L` is at most

\[
\sum_{j=0}^{L-1}A^j
\le \frac{A^L}{A-1}.
\]

### Theorem 2 — typical explicit lookup complexity

Choose an exceptional fraction `epsilon` in `(0,1)`. If

\[
\frac{A^L}{A-1}\le\epsilon 2^B,
\]

then at most an `epsilon` fraction of all `N`-entry, `w`-bit tables can be
implemented by circuits shorter than `L`. Consequently at least a
`1-epsilon` fraction require at least `L` gates under the declared model.

### Proof

Every circuit implements at most one lookup unitary. The number of lookup tables
admitting a circuit shorter than `L` is therefore no larger than the number of
such circuits. Divide the geometric-series upper bound by `2^B`. \(\square\)

The executable helper returns the conservative integer

\[
L=\left\lfloor
\frac{B+\log_2\epsilon+\log_2(A-1)}{\log_2 A}
\right\rfloor_+.
\]

This is a typical-family or worst-case existence result, not a universal lower
bound for every table. Constant, affine, product-domain, grammar-generated, and
model-generated tables can have much smaller reversible descriptions.

## 5. Auditable minterm upper bound

For an independent exact baseline, a literal lookup can be synthesized by
iterating over addresses:

1. conjugate zero-valued address controls by X gates;
2. apply one multi-controlled X to each output bit equal to one;
3. undo the negative-control X gates.

With `m` address controls and `m-2` clean ancillas, the standard conjunction
construction uses `2m-3` Toffoli gates per output-one bit when `m>=2`. The
executable resource counter reports:

- address and word widths;
- table description bits;
- nonzero entries and output-one bits;
- X, CNOT, and Toffoli counts;
- clean ancillas and total logical qubits.

This minterm construction is an upper bound and an independent correctness
backend. It is not an efficient solution for arbitrary large empirical datasets.

## 6. Amortization condition

Let the compiled empirical lookup be reused for `M` reconstruction tasks. In one
common cost unit, write

\[
C_C(M)=S_C+M V_C,
\]

\[
C_Q(M)=S_Q+M V_Q.
\]

For an explicit table, the quantum setup must include at least the declared cost
of consuming the `B` table bits unless QRAM/preloading is an explicit external
assumption. If `V_Q<V_C`, strict advantage requires

\[
M>\frac{S_Q-S_C}{V_C-V_Q}.
\]

If `V_Q>=V_C` and `S_Q>=S_C`, no amount of reuse produces an advantage under
this model.

The executable amortization helper takes separate classical and quantum setup
costs per table bit plus extra setup and variable costs. A zero quantum setup
factor is allowed only to make a QRAM or already-compiled-oracle assumption
visible in the report.

## 7. Consequence for real-data candidate sets

A candidate set copied verbatim from GIFT-Eval, Community Forensics, or another
corpus is an explicit empirical table unless the experiment supplies a shorter
reversible generator. Therefore a valid end-to-end report must choose one of the
following contracts.

### Contract A — explicit compilation

Count dataset reading, quantization, lookup-circuit synthesis, storage, logical
qubits, and every lookup/inverse call. Report the number of reconstruction tasks
sharing that setup.

### Contract B — physical QRAM

Specify the QRAM architecture, load/update cost, capacity, fault-tolerant error
model, access latency, and whether the classical baseline receives an equivalent
indexed data structure. “Assume QRAM” is not a resource report.

### Contract C — succinct structured generator

Provide a reversible program whose description is sublinear in the explicit
candidate table and count its execution and inverse. Product domains, arithmetic
constraints, grammars, and compact generative priors belong here. The verifier
must still report collisions and the target equivalence.

### Contract D — no coherent data loading

Restrict the claim to classical C/W-Access or to an information-theoretic bound.
A quantum computer choosing classical queries does not acquire `U_T`.

## 8. Claim consequences

The following statements are invalid without an explicit loading contract:

- “Grover searches the real dataset in `O(sqrt(N))` time.”
- “The dataset is already available, so oracle construction is free.”
- “Classical preprocessing is counted but QRAM loading is not.”
- “A truth-table circuit proves scalable quantum advantage.”

A defensible statement is narrower:

> Conditional on a declared coherent lookup implementation and its amortized
> setup, the downstream unstructured search uses fewer verifier calls; the
> end-to-end phase diagram determines whether that reduction survives loading,
> verification, fault tolerance, and the strongest matched classical solver.

## 9. Theorem-to-code mapping

| Result | Executable implementation |
|---|---|
| explicit table dimensions | `explicit_lookup_description` |
| Theorem 1 | `explicit_table_compiler_bit_probe_lower_bound` |
| Theorem 2 | `typical_lookup_circuit_lower_bound` |
| minterm upper bound | `lookup_minterm_resources` |
| amortization condition | `explicit_lookup_amortization_report` |

Focused tests cover non-power-of-two dimensions, exact small-table gate counts,
monotonic growth of the counting bound, weakening under extra workspace,
amortization thresholds, and invalid inputs.

## 10. Next integration milestone

The real-data benchmark should emit, for each candidate prior:

- explicit table bits and source-data hash;
- whether the candidate set is table-based or generated;
- lookup/state-preparation contract;
- typical counting lower bound and exact synthesized resources when tractable;
- reusable setup count;
- matched classical index/preprocessing cost;
- end-to-end sensitivity to QRAM and no-QRAM assumptions.

This turns data access from an implicit convenience into a falsifiable component
of the final advantage or no-advantage result.
